# Character Sprites

Community-contributed character sprite sheets for Pixel Office.

## Format

- **PNG**, RGBA, transparent background.
- **112 × 96 pixels** total — a 7 × 3 grid of 16 × 32 frames.
- Frame layout (per row):

  | col 0  | col 1 | col 2  | col 3  | col 4  | col 5  | col 6  |
  |--------|-------|--------|--------|--------|--------|--------|
  | walk_A | idle  | walk_B | type_A | type_B | read_A | read_B |

  - Row 1: facing DOWN (face visible)
  - Row 2: facing UP (back view)
  - Row 3: facing RIGHT (side profile, one eye dot visible)
- LEFT direction is mirrored from RIGHT at runtime — do not draw.

## Directory layout

```
sprites/characters/
  pink_hoodie/
    sprite.png       # the 112×96 sheet
    metadata.json    # name, author, description, tags, date
    thumbnail.png    # 16×32 idle pose (auto-generated by CI)
```

## metadata.json schema

```jsonc
{
  "name": "Pink Hoodie",

  "issueNumber": 7,
  "votes": 0
}
```

## Generation tips

- Each frame is 16 × 32 px; the character occupies the **bottom 24 px** with the top 8 px reserved for hair tufts / breathing space.
- All 21 frames must use the SAME palette (~6–8 colors) so the character looks identical from every angle.
- `walk_A` and `walk_B` should have **visibly different leg positions** — mirror images, not the same pose.
- Prompt template lives in the main repo at `pixel-office-assets/characters/PROMPTS.md`.
