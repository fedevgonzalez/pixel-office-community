# Pet Sprites

Community-contributed pet sprite sheets for Pixel Office.

## Format

- **PNG**, RGBA, transparent background.
- **160 × 96 pixels** total — a 5 × 3 grid of 32 × 32 frames.
- Frame layout (per row):

  | col 0  | col 1  | col 2 | col 3  | col 4  |
  |--------|--------|-------|--------|--------|
  | walk_A | walk_B | idle  | sleep_A| sleep_B|

  - Row 1: facing DOWN (towards camera, face visible)
  - Row 2: facing UP (back view, no face)
  - Row 3: facing RIGHT (side profile, one eye dot visible)
- LEFT direction is mirrored from RIGHT at runtime — do not draw.

## Directory layout

Each variant lives in its own subdirectory:

```
sprites/pets/
  cat_calico/
    sprite.png       # the 160×96 sheet
    metadata.json    # name, author, species, description, tags, date
    thumbnail.png    # 32×32 idle pose (auto-generated by CI)
```

## metadata.json schema

```jsonc
{
  "name": "Pumpkin",                // display name of the breed/variant
  "species": "cat",                 // "cat" | "dog"
  "author": "fedevgonzalez",
  "description": "Orange-and-white calico with bright copper eyes.",
  "tags": ["cat", "calico", "warm"],
  "createdAt": "2026-05-17",
  "issueNumber": 42,                // populated by CI on submission
  "votes": 0
}
```

## How to submit

See [../../CONTRIBUTING.md](../../CONTRIBUTING.md). TL;DR: drop a directory with the three files above, open a PR, the bot validates dimensions and generates a thumbnail.

## Generation tips

- Use ChatGPT image gen, not Gemini (Gemini bakes the transparency-checker pattern as opaque pixels).
- The pixel-office repo ships `scripts/integrate-sprite-sheet.mjs`, which cleans AI output (anti-aliasing → crisp pixels, color quantization, transparency detection) via [`proper-pixel-art`](https://github.com/KennethJAllen/proper-pixel-art).
- Prompt template lives in the main repo at `pixel-office-assets/pets/PROMPTS.md`.
